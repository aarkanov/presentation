  • Appium is an open source test automatiom framework for use with native and hybrid mobile apps
  • It drives iOS and Android apps using the WebDriver JSON Wire Protocol
  • Large community
  • Frequent releases
  • lots of support
  • allowed us to leverage our Selenium knowledge

  • The other tool we used was developed specifically for AngularJS
  • Protractor is an end-to-end test framework
  • It runs tests against your application running in a real browser
  • It interacts as user would
  • Since our app was hybrid, using protractor allowed us to run multiple functional tests in a fast manner
  • while running sanity tests on real devices
  • Monkey test was another approach we had to make sure our apps were stable
  • A monkey test runs with no specific test in mind
  • It would enter random strings into boxes
  • Click on random boxes
  • Open and close the app
  • Attempt to open other apps and then our app
  • Purpose was to make the app crash through stranious use

Devices

  • For iOS we decided that we will support iPhone 4,5 and iPad 2 and up
  • We currently have iPhone 4,5,3gs and iPad 2 as test devices
  • For Android we needed a bit of a different approach due the vast spectrum of devices and OS
  • We decided we will support OS 4.0 and up
  • In order not to purchase 200 devices, we chose to have a low end and a high end device
  • and make the assumption that if the app works on those, it will work on everything in between
  • Once all the devices arrived we had to connect them and set them all up for automation
  • This is what the Appium setup looks like
  • We use Git to store all our test code
  • Bamboo to allow us to run the tests whenever we want
  • We have a selenium server that allows us to have multiple devices connected to appium
  • and we have the appium servers running on the Mac machines that correspond to every device
  • This is what it actually looks like

Future of Mobile Testing at OTPP

  • So where do we go from here??
  • If we continue developing apps in the future they will get more complicated
  • We will have lots of components that will have to be tested
  • And automation will be essential to make sure that we can tests things quickly
  • The new website is a 5 year project and we will have to make sure it meets its mobile first objective
  • We will leverage all the tools that we have used so far to ensure succesfull testing on the new website
  • Mobile and Web security issues are in the news on a daily basis
  • We will have to make sure that our connected apps, and our mobile sites are secure
  • We will also have to make sure that the member is informed regarding security issues
  • More browser support
  • We have users that are using pcs/macs/tablets/phones
  • Android/iOS/Windows/Linux/Mac
  • We need to make sure that our website are working, and are displaying properly
  • on any device that a user logs in on

  • In order to have access to all browsers we need to use tools such as Sauce labs
  • Allows us to test across multiple browsers, and multiple operating systems
  • They will also have a future open of testing on real devices over the cloud
  • Performance testing will also be very important
  • Most users these days have very high expecations when it comes to web and applications
  • We will have to introduce new tools and make sure that our users get the optimal performance experience
  • By 2017 most people will have at least 5 devices that are connected to the internet
  • Most of you in the room most likely have at least 2
  • We will have to make sure that the experience is the same accross all devices
